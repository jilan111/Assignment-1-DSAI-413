=======================================================
           سكريبت الديمو — DocMind RAG
=======================================================

[مقدمة — 30 ثانية]
-----------------------
أهلاً بيكم! 

النهارده هنقدم ليكم نظام DocMind، اللي هو نظام سؤال وجواب ذكي مبني على تقنية الـ RAG — أو "الاسترجاع المُعزَّز بالتوليد".

المشكلة اللي بيحلها النظام ده: إيه اللي تعمله لما يكون عندك ملف PDF كبير وعايز تسأل أسئلة عنه من غير ما تقرا كل صفحة بنفسك؟

الحل هو DocMind!


[شرح النظام — 60 ثانية]
--------------------------
النظام بيشتغل على أربع خطوات:

1️⃣ الاستيعاب (Ingestion):
   النظام بياخد الـ PDF ويستخرج منه النصوص، الجداول، والصور.

2️⃣ التقطيع الذكي (Chunking):
   المحتوى بيتقسم لأجزاء صغيرة ذكية مع تداخل بين الأجزاء عشان منضيعش معلومة.

3️⃣ الـ Embeddings والبحث (Retrieval):
   كل جزء بيتحوّل لـ vector رقمي، ولما تسأل سؤال، النظام بيجيب أقرب 5 أجزاء للسؤال ده.

4️⃣ توليد الإجابة (QA Generation):
   اللغة الكبيرة (LLM) بتستخدم الأجزاء دي وتركّب إجابة كاملة مع ذكر المصادر.


[الديمو الحي — 90 ثانية]
--------------------------
تعالوا نشوف النظام بيشتغل!

الخطوة 1: بنفتح التطبيق — هتلاقوا واجهة جميلة بتصميم الـ glassmorphism.

الخطوة 2: بنضيف الـ API key في الـ sidebar من اليسار. لو الـ key غلط، النظام مش هيعطل، هيقولك: "Invalid API key. Please verify and try again."

الخطوة 3: بنرفع الـ PDF — مثلاً ملف أكاديمي أو تقرير عمل.

الخطوة 4: بنضغط "Process Document" — والنظام بيقوم بعمل الـ indexing.

الخطوة 5: بنسأل أسئلة! مثلاً:
   - "ما هو الهدف الرئيسي من هذه الدراسة؟"
   - "ما هي النتائج الرئيسية؟"
   - "اشرح الجداول الموجودة في الصفحة الثالثة."

[المميزات الرئيسية — 45 ثانية]
----------------------------------
✦ يشتغل مع أي ملف PDF — مش محدود بموضوع معين
✦ يدعم أي API متوافق مع OpenAI — تقدر تغير للـ Groq أو Together بسهولة
✦ تصميم أنيق وسهل الاستخدام
✦ عرض المصادر بشكل واضح مع كل إجابة
✦ معالجة شاملة للأخطاء — مفيش crash أبداً


[الخاتمة — 20 ثانية]
-----------------------
النظام ده بيحل مشكلة حقيقية: استخلاص المعلومات من المستندات الطويلة بسرعة ودقة.

الكود كامل، مُختبر، وجاهز للتسليم.

شكراً ليكم! 🌸
